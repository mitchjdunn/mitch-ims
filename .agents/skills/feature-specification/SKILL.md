# Feature Specification Skill

Use this skill when drafting new specifications inside `/docs/features/`.

## Core Directives
1. **Standard Format**: Ensure all feature specs follow the template structure: Overview, User Stories, UI/UX demands, REST API endpoints, DB Schema requirements, and QA checklists.
2. **Continuity**: Cross-reference existing files in `docs/` and database schemas to preserve design consistency.
